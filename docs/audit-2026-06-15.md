# Security, Performance & Clean Code Audit — 2026-06-15

Branch: `claude/youthful-hypatia-aazq5y` (merges `dev`)

## Summary

Full audit of the OBRS-frontend Angular 18 codebase. All fixes were applied on the designated branch and verified with `ng build --configuration production` (zero errors, only pre-existing budget warnings).

---

## 1. Security Fixes

### 1.1 Console log of sensitive data — `otp-validate.component.ts`
- **Removed** `console.log(resVerify)` — was printing the full OTP verification response (may contain auth tokens).
- **Removed** `console.log(registerValue)` — was printing user registration payload.

### 1.2 Deprecated `.toPromise()` → `firstValueFrom()` — `otp.service.ts`
- `requestOTP()` and `verifyOTP()` used the deprecated `.toPromise()` pattern, which is removed in RxJS 8 and against project conventions.
- Replaced both with `firstValueFrom()` from `rxjs`.

---

## 2. Memory Leak Fixes (`renderer.listen()` unlisten pattern)

`Renderer2.listen()` returns an unlisten function that must be stored and called in `ngOnDestroy`. All instances below were either missing the call or not storing the return value.

| Component | Fix |
|---|---|
| `login.component.ts` | Added `unlistenDropdown?: () => void`, stores return value, calls in `ngOnDestroy` |
| `login-mobile.component.ts` | Same |
| `forget-password.component.ts` | Same |
| `register.component.ts` | Same |
| `navbar.component.ts` | Added two unlisten fields (language + profile dropdowns) |
| `dropdown-group-obrs.component.ts` | Added `OnDestroy`, unlisten field, `ngOnDestroy` |
| `dropdown-obrs.component.ts` | Same |
| `dropdown-obrs-passenger.component.ts` | **Bug fix**: field was declared but return value was never assigned; fixed assignment + `ngOnDestroy` |
| `otp-validate.component.ts` | Added unlisten field, stores and calls in `ngOnDestroy` |

---

## 3. Subscription Accumulation Fix

### `schedule-booking-list.component.ts`
- `selectSchedule()` was creating a new `.subscribe()` on every call without unsubscribing the previous one.
- Fixed: unsubscribe before reassigning, use `pipe(take(1))`.

## 6. JSON Syntax Error — `public/i18n/zh.json`

- **Critical bug**: `PRIVACY.CONTENT_2` contained unescaped ASCII double-quote characters (`"个人数据"`), causing a JSON parse error at line 634.
- The entire Chinese locale file failed to load, breaking all `zh` users.
- **Fixed** by escaping the quotes: `\"个人数据\"`.

## 9. TypeScript `any` Elimination

### `passenger-info.component.ts`
- `isReturnTrip(roundTrip: any)` → typed as `ScheduleFilter['roundTrip'] | number | null | undefined`
- `extractBookingId(data: any)` → typed as `BookingCreationResponse | null | undefined`
- `extractBookingNumber(data: any)` → typed as `BookingCreationResponse | null | undefined`

### New interface: `BookingCreationResponse` (`booking.interface.ts`)
- Captures all possible fields the backend returns on booking creation (`bookingId`, `id`, `bookingNumber`, `bookingNo`, `number`, nested `booking`).

### `booking.service.ts`
- `createBooking()` return type changed from `Observable<ResponseAPI<any>>` to `Observable<ResponseAPI<BookingCreationResponse>>`.
