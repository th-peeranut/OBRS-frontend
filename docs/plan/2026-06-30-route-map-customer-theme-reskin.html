Route-Map Customer-Theme Re-skin Implementation Plan

เป้าหมาย: Re-skin เฉพาะส่วนแผนที่เส้นทาง (route-map-home + คอมโพเนนต์ลูก) บนหน้า /home ให้เข้าธีม customer (cyan-blue) แทนสไตล์ default ของ PrimeNG/Bootstrap ที่หลุดธีมและอ่านยาก โดยคงเลย์เอาต์และฟังก์ชันเดิมทั้งหมด

สถาปัตยกรรม: เปลี่ยนเฉพาะเลเยอร์ภาพ — ค่าสีใน SCSS, ค่าสี marker/polyline ใน TypeScript, และ class ใน template ของ 5 คอมโพเนนต์ใต้ home/components/route-map/. ดึง design token จาก src/styles/variables.scss ที่มีอยู่แล้ว ทับสไตล์ PrimeNG ผ่าน :host ::ng-deep แบบ scoped. ไม่แตะ logic, NgRx, API, layout (3-คอลัมน์/3-แท็บ) หรือ Google Map

Tech Stack: Angular 18 (module-based), SCSS component-scoped, PrimeNG 17, Bootstrap 5 (utilities), Jasmine + Karma (ChromeHeadless)

For agentic workers: REQUIRED SUB-SKILL: Use thaibev-subagent-driven-development (recommended) or thaibev-executing-plans to implement this plan task-by-task. Steps keep the - [ ] marker next to each checkbox for tracking.

Global Constraints

  • ทุกคำสั่ง terminal รันจากโฟลเดอร์ OBRS-frontend/ (ไม่ใช่ repo root)
  • เปลี่ยนเฉพาะเลเยอร์ภาพ — ห้ามแตะ layout 3-คอลัมน์/3-แท็บ, Google Map logic, NgRx, API, และ flow auth/payment/booking
  • ใช้สีจาก design token ใน src/styles/variables.scss เท่านั้น (ห้าม hardcode hex สุ่ม); token หลักที่ใช้:
    $primary-blue#4bc2f7= สี pickup (จุดรับ)
    $secondary-blue#4069b8= สี dropoff (จุดส่ง)
    $secondary-lightgrey#edf9fe= พื้น section
    $secondary-light-blue#d9e1f1= เส้นขอบการ์ด
    $text-black#353c44= หัวข้อ
    $text-softblack#535968= เนื้อหา
    $text-lightblack#757a86= ป้าย/meta
    $text-primary#4069B8= tab/ลิงก์ active
    $text-red#cb393a= ไอคอน error เท่านั้น
  • โทน monochrome ฟ้าล้วน: pickup = $primary-blue, dropoff = $secondary-blue; แยกความต่างด้วยไอคอน (bi-geo-alt รับ / bi-geo-alt-fill ส่ง) + ป้าย "จุดรับ/จุดส่ง" + อ่อน-เข้มของฟ้า ไม่พึ่งเฉดสีอย่างเดียว
  • การ์ดทุกใบ: ขอบมน 24px (เท่าการ์ดค้นหาด้านบน), เงา 0 8px 24px rgba(64,105,184,.08), ขอบ 1px solid $secondary-light-blue
  • ตัวอักษรขั้นต่ำ 0.875rem สำหรับเนื้อหา และ 0.8125rem สำหรับ meta (เลิกใช้ 0.75rem); แทน Bootstrap .text-muted ที่ลอย ๆ ด้วยสี token จริง
  • ทับสไตล์ PrimeNG ผ่าน :host ::ng-deep แบบ scoped ในไฟล์ .scss ของคอมโพเนนต์นั้นเท่านั้น (ห้ามไปแก้ global)
  • ไม่มี i18n key ใหม่ (เปลี่ยนแค่ภาพ) — ไม่ต้องแตะ public/i18n/*.json
  • Path import variables ในคอมโพเนนต์ใต้ route-map/ (ลึก 6 ชั้น) คือ @import "../../../../../../styles/variables.scss"; (หก ../)
  • เฝ้า budget: component style warning ที่ 6kB / error ที่ 10kB ต่อไฟล์ (ระบุใน CLAUDE.md §7)
  • Risk = R2 (UI ล้วน) — commit ได้ทันที, แต่ ห้าม push เอง เว้นแต่ผู้ใช้สั่ง
  • Type gate ที่เชื่อถือได้คือ npx tsc --noEmit -p tsconfig.app.json (dev build เขียว ≠ type ผ่าน ตาม CLAUDE.md §10)
